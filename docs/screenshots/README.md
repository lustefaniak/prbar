# PRBar screenshots

Generated by `bin/screenshots` (which runs `Tests/PRBarTests/ScreenshotTests.swift` via `ImageRenderer @ 2x`). Mock fixtures live in the same test file — there's no `gh` / `claude` / network involvement, so anyone can regenerate these reproducibly.

## Regenerate

```sh
bin/screenshots
```

## Use as UI regression net

Re-running rewrites every PNG in place. After a UI change, run the script and `git diff docs/screenshots/` — pixel-level diffs in the GitHub PR view show what visually changed and let a reviewer catch unintended regressions.

## What's covered

| File | Scene |
| --- | --- |
| `01-inbox-list@2x.png` | Inbox tab — review-requested PRs with AI verdicts inline |
| `02-my-prs@2x.png` | My PRs — ready-to-merge rows show the green Squash button |
| `03-pr-detail-completed@2x.png` | Detail view with completed AI review (verdict, summary, annotations, diff, actions) |
| `04-pr-detail-running@2x.png` | Detail view mid-triage — live tool/cost progress visible |
| `05-pr-detail-retriage@2x.png` | Detail view during retriage — prior verdict shown alongside live progress |

## What's not covered

Settings panes (`General`, `Repositories`, `Diagnostics`, `About`) are intentionally absent — SwiftUI `Form` controls (`Toggle`, `Slider`, `TextField`, `Picker`, `Stepper`, `TextEditor`) are `NSControl`-backed, and `ImageRenderer` captures them as a yellow "image-not-loaded" placeholder. Capture those manually with `screencapture -wo path.png` against a running app when needed.

## How the rendering works

`ScreenshotTests` uses `ImageRenderer` from SwiftUI (macOS 13+). For each scenario it builds `@MainActor @Observable` mock services (`PRPoller`, `ReviewQueueWorker`, `DiffStore`, `ReadinessCoordinator`) seeded via `_setForScreenshot(...)` helpers, injects them via `.environment(...)`, then renders the view at scale 2.0 to a PNG.

A few production views grow a `screenshotMode: Bool = false` parameter: `PRDetailView`, `PRRowView`, `PRListView`, `RepoConfigEditor`. When true, the view swaps `ScrollView` for a flat `VStack` (so all sections render in one frame instead of being clipped) and replaces `Menu` with plain `Button`s (which `ImageRenderer` can capture). Production callers leave the flag at its default and see no change.
