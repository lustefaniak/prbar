# Claude Code working notes for PRBar

## What this repo is

PRBar — macOS menu-bar Swift app that monitors my GitHub PRs (via `gh`) and runs AI-assisted reviews on incoming review requests (via `claude` CLI). Full design in [docs/PLAN.md](docs/PLAN.md). Read it before making non-trivial changes.

## Build / test / run

ALWAYS use `bin/` wrappers. Don't invoke `xcodegen` or `xcodebuild` directly.

```sh
bin/regen   # regenerate PRBar.xcodeproj from project.yml
bin/build   # bin/regen + xcodebuild build
bin/test    # bin/regen + xcodebuild test
bin/run     # bin/build + open .app (kills prior instance first)
```

When you change `project.yml`, file layout, or add a new Swift file under `Sources/`, run `bin/regen` (or `bin/build`, which regenerates first).

## Project structure

- `project.yml` — XcodeGen spec. Single source of truth for targets, build settings, Info.plist keys.
- `Sources/PRBar/` — Swift sources, grouped by responsibility (UI, Services, Models, Util).
- `Resources/` — bundled assets (prompts, JSON schemas, default monorepo configs).
- `Tests/PRBarTests/` — XCTest cases. Pure-Swift logic (DiffParser, MonorepoSplitter, etc.) gets covered here.
- `bin/` — wrapper scripts. Add new automation here, not as ad-hoc xcodebuild invocations.
- `PRBar.xcodeproj/` — **gitignored**. Generated by xcodegen. Never edit by hand.

## Conventions

- **Swift 6 strict concurrency** (`SWIFT_STRICT_CONCURRENCY = complete`). Use actors for mutable shared state, `Sendable` types across boundaries.
- **No third-party deps in Phase 0–4.** If you reach for one, raise the trade-off explicitly first.
- **Subprocess execution**: Phase 0 uses `Foundation.Process`. Phase 1+ migrates to `swift-subprocess` (Apple, Swift 6.2+). Don't mix the two within a single feature.
- **No comments unless WHY is non-obvious.** Don't restate code in comments.
- **No emojis in code, commits, or UI strings** unless explicitly requested.

## Phases

- Phase 0: Skeleton (current) — menu-bar app, tool-availability smoke check, launch-at-login toggle.
- Phase 1: PR poller + UI rows + manual merge action (no AI).
- Phase 2: Minimal-tools AI review pipeline.
- Phase 3: Diff view with annotations.
- Phase 4: Monorepo splitting + per-subfolder reviews.
- Phase 5: Pure-prompt mode (opt-in).
- Phase 6: Auto-approve policy + 30s undo.
- Phase 7: Polish.

## Known caveats (this Mac)

- `xcodebuild` requires full Xcode, not Command Line Tools. If `xcode-select -p` returns `/Library/Developer/CommandLineTools`, run `sudo xcode-select --switch /Applications/Xcode.app/Contents/Developer`.
- The app is unsandboxed and ad-hoc-signed for local dev — required to fork/exec `gh`, `claude`, `git`.
- `SMAppService.mainApp.register()` for launch-at-login works fully only when the app lives in `/Applications`. From a Debug build it'll throw a not-fatal warning; ignore it during dev.

## Don't

- Don't commit `PRBar.xcodeproj/` — it's generated.
- Don't add Swift files via Xcode "Add Files…" — drop them in `Sources/PRBar/<subdir>/` and run `bin/regen`.
- Don't push directly to `main` — feature branches only.
- Don't call `claude` with `--permission-mode default` or `bypassPermissions` from app code — see plan §"AI Review Pipeline" for the locked invocation shape.
- Don't add Bash/Edit/Write tools to the AI's `--allowedTools` list — those are deliberately disabled.
